# starter-api — project-specific правила

- **Шаблон публичный** — не содержит секретов и prod-конфигов.
- **Не снижать порог coverage** ниже 100%.
- **In-memory store — пример** — заменить на реальную БД при клонировании.
- **Три уровня тестов обязательны** — unit, integration (vitest), E2E (Playwright).
